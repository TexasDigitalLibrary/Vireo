Detailed Description

Search indexer.

The purpose of this interface is to receive updates to various model objects and update the search index. Typically the implementation of this interface is also tied to the implementation of the sister interface: Searcher.

void org.tdl.vireo.search.Indexer.deleteAndRebuild ( boolean  wait)

Delete the entire search index and rebuild it from scratch. This will cancel any index jobs currently in progress, and the index will be irrevocably deleted. Following the deletion, the index will be rebuild.

While this index is rebuilding searches may not be performed against the index. Any search that does process will result in an exception.

Parameters
waitIf true the method will not return until rebuilding the index is completed, otherwise rebuilding will take place in a background thread.

public<T extends AbstractModel> void org.tdl.vireo.search.Indexer.updated ( T  model)

This is a notification method that the provided model object has been updated (either created, saved, or deleted), but has not been committed to the database yet. The indexer will keep this proposed update until either rollback() or commit() is called from the same thread.

Parameters
modelThe model object which was created, saved, or deleted.

void org.tdl.vireo.search.Indexer.updated ( Long  submissionId)

This is a notification method to update the index for the provided submission Id. This works the same as the updated(model) varient however this may be more efficient in situations where the full object is not available. The indexer will keep track of these submissions until either rollback() or commit() is called from the same thread.

Parameters
submissionIdThe id of the submission which was created, saved, or deleted.
