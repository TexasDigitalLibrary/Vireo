Detailed Description

Explicit authentication method used when Vireo challenges the user for a username and password. Implementors must implement either Implicit or Explicit authentication interfaces.

AuthenticationResult org.tdl.vireo.security.AuthenticationMethod.Explicit.authenticate ( String  username,
String  password,
Request  request 
)

Authenticate these credentials. If valid credentials have been presented then the user will be logged into Vireo's SecurityContext, and a AuthenticationResult.SUCCESSFULL result.

Note: The interface does not include a security realm, if that is needed for this authentication method it should be combined with the username, thus REALM\username or something similar to how microsoft handles the case.

If authentication is anything other than successful then one of the failure result codes should be returned.

MISSING_CREDENTIALS: Either the username or password were missing.

BAD_CREDENTIALS: The username and password are present but they do not match any valid user.

UNKNOWN_FAILURE: This is the catch all case for unforeseen errors conditions.

Parameters
usernameThe username to be authenticated.
passwordThe password to be authenticated.
