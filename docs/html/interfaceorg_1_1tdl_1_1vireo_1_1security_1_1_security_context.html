Detailed Description

This singleton interface keeps track of the current security context that Vireo is operating within.

When a user is authenticated by the UI the security context is notified via the login() method. The login will be kept track of (using a ThreadLocal-based mechanism) of the currently authenticated user. This means that whenever a component needs to access the currently authenticated user they can just query the current security context.

Examples:

SecurityContext context = Spring.getBeanOfType(SecurityContext.class);

Person authenticated = context.getPerson();

Or if you just need to make a quick decision based upon the person's current role you can ask one of the isRole methods such as:

if (context.isAdministrator()) { // Do something very special. }
